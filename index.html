01 · The broken model

Every session starts cold. Decisions, preferences, and hard-won project knowledge vanish when the window closes.

Stuffing the context window doesn't scale — it fills up. Flat markdown files go stale and nobody prunes them. Vector databases find text that looks similar but have no concept of how memories relate to each other, so an isolated note outranks the decision it contradicts. Agents need a graph, not a bigger bucket.

03 · The system

From long-term memory out to the tooling around it — every layer has a job.

menhir launch The memory
Graph-based long-term memory for AI agents, built on Neo4j and Graphiti and exposed as an MCP server. Memories are entities with relationships, and those relationships count during retrieval — a well-connected memory outranks an isolated note with the same keywords. It also indexes your project structure into the same database, so recall is code-aware and any file change comes with a blast radius.
Owns
  • Cross-session entity graph
  • Four-signal recall scoring
  • Code graph + blast radius
  • Decay, compression, conflicts
archolith-filter alpha The filter
Token Reduction Toolkit — deterministic output filtering for LLM agent contexts. Layer 0 pre-filter pipeline (secrets, thinking, paths, binary, oversized) plus nine format-switch strategies for diffs, search, file reads, JSON, logs, stack traces, and build output. 50% average savings on real session data.
Owns
  • Layer 0 pre-filter pipeline
  • Nine format-switch strategies
  • Token estimation + risk presets
  • Cross-turn deduplication
archolith-skree alpha The diagnostics
MCP token usage audit system. Measures per-server token cost, detects waste patterns, and produces report cards with concrete optimization suggestions.
Owns
  • Server attribution
  • Waste detection (6 types)
  • Report cards
  • CI threshold gates
archolith-peira benchmark The yardstick
Reproducible benchmark suite measuring recall quality, filter compression, and MCP waste reduction across multi-turn coding scenarios. Public claims stay tied to tracked evidence artifacts.
Owns
  • Recall scenario runner
  • Filter corpus compression
  • Audit before/after
  • Stack comparison arms
archolith-context coming The proxy
Research direction for curated context assembly at the request boundary. An OpenAI-compatible proxy would query the memory graph each turn and splice curated context in place of linear replay, with no harness code changes. Still experimental and not part of the launch.
Would own
  • /v1/chat/completions
  • SSE pass-through
  • Curator assembly
  • Trace API + dashboard

05 · Install

menhir is self-hosted — Neo4j, an LLM backend, and your MCP client. Public install lands with the release.

In the meantime, archolith-skree installs today and measures where your MCP context is going.

beta

Native Claude plugin install with managed Python runtime bootstrap.

beta

Clones the public Codex plugin and registers the audit MCP server.

curl -fsSL https://raw.githubusercontent.com/Archolith/archolith-mcp-audit/master/scripts/install.sh | bash -s -- codex
OpenCode
beta

Clones the public OpenCode plugin and writes the local plugin + MCP config.

curl -fsSL https://raw.githubusercontent.com/Archolith/archolith-mcp-audit/master/scripts/install.sh | bash -s -- opencode

Each install keeps runtime dependencies in an isolated venv under ~/.archolith/venvs/. Gemini CLI is omitted because that CLI is deprecated.

06 · Run it

Five lines to running.

$ docker run -d -p 7687:7687 -e NEO4J_AUTH=neo4j/password neo4j:5 $ pip install -e ".[dev]" $ cp .env.example .env # Neo4j + LLM backend settings $ python -m menhir # MCP server over stdio $ menhir serve # graph explorer on :8090
